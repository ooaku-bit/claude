# Skill Compose Skill

## 目的

蓄積されたナレッジを活用して、新しいスキルを作成するか、既存スキルを継続的に改善する。

## トリガー条件

使う場面:
- 繰り返し登場する作業パターンをスキル化したい
- `skill-learnings/` に改善フィードバックが蓄積されている
- `/skill-new` コマンドが呼ばれた

使わない場面:
- 一度きりのタスク（スキル化の価値がない）
- 既存スキルで十分対応できる

## 手順（新規作成）

1. **ユースケース定義**: どんな問題を解くスキルか
2. **既存スキル確認**: `REGISTRY.md` で重複がないか確認
3. **テンプレートコピー**: `_template/SKILL.md` をベースにする
4. **フォルダ作成**: `.claude/skills/<スキル名>/SKILL.md`
5. **REGISTRY更新**: `skills/REGISTRY.md` に追記

## 手順（既存スキル改善）

1. **学習ログ確認**: `skill-learnings/README.md` で対象スキルの知見を収集
2. **現スキル確認**: `skills/<スキル名>/SKILL.md` を Read
3. **差分特定**: 何を変えるべきか明確にする
4. **Edit適用**: 変更を加える（バージョン番号を上げる）
5. **変更履歴更新**: SKILL.md 内の変更履歴テーブルを更新
6. **REGISTRY更新**: バージョン・日付・評価を更新

## スキル品質チェックリスト

- [ ] 名前が動詞または名詞で明確
- [ ] description にトリガー条件が含まれる
- [ ] 手順が具体的で再現可能
- [ ] 成功基準が測定可能
- [ ] 200行未満に収まっている

## 出力フォーマット

```
## スキル作成完了: <スキル名>

- 場所: .claude/skills/<スキル名>/SKILL.md
- バージョン: X.Y
- REGISTRY: 更新済み
```
