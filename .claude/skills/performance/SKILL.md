# Performance Skill

## 原則

- **計測なき最適化はしない**: 推測でなく数値で判断する
- **ボトルネックから潰す**: 全体の80%を占める20%を先に直す
- **改善前後を比較する**: ベースラインを必ず記録する

## 分析ステップ

1. **プロファイリング**: ホットスポットを特定する
   ```bash
   # Python
   python -m cProfile -s cumulative script.py
   # Node.js
   node --prof script.js && node --prof-process isolate-*.log
   ```

2. **ベースライン計測**: 改善前の数値を記録する

3. **ボトルネック分析**: 以下の観点で確認する
   - N+1クエリ（ループ内DB呼び出し）
   - 不要な繰り返し計算（キャッシュ候補）
   - 大きなデータ構造のコピー
   - 同期I/Oのブロック
   - アルゴリズムの計算量（O(n²) など）

4. **改善実装**: 1箇所ずつ変更する

5. **再計測**: 改善効果を数値で確認する

## チェックリスト

- [ ] ベースライン数値を記録した
- [ ] プロファイラでホットスポットを特定した
- [ ] 改善後に同条件で再計測した
- [ ] 機能の正確性が損なわれていない
