# Knowledge Capture Skill

## 目的

タスク・セッション完了後に「何を学んだか」を構造化して記録し、次のセッションで活用できる組織知識を積み上げる。

## トリガー条件

使う場面:
- タスクが完了した直後
- 予想外の問題に遭遇・解決したとき
- 設計上の重要な決定を下したとき
- 失敗して原因を突き止めたとき

使わない場面:
- 単純な質問への回答
- 既にナレッジとして記録済みの内容

## 手順

1. **タスク内容を要約**: 何をやったか、1〜3文
2. **学びを分類**:
   - 再利用できる → `patterns/PATTERNS.md`
   - 設計上の決定 → `decisions/ADR-NNNN-*.md`
   - スキルへのフィードバック → `skill-learnings/README.md`
3. **記録**: 適切なファイルに追記
4. **INDEX更新**: `knowledge/INDEX.md` の統計カウントを更新
5. **REGISTRY更新**: スキルの評価を更新（必要なら）

## 記録フォーマット

### パターンの場合

```markdown
### [パターン名]
**状況**: 
**解決策**: 
**注意**: 
**出典**: YYYY-MM-DD タスク名
```

### ADRの場合

ファイル名: `ADR-NNNN-タイトル.md`（NNNNは連番）

### スキル学習の場合

```markdown
## <スキル名> — YYYY-MM-DD

**コンテキスト**: 
**うまくいったこと**: 
**うまくいかなかったこと**: 
**提案する変更**: 
```

## 成功基準

- [ ] 学びが適切なカテゴリに記録された
- [ ] INDEX.md の統計が更新された
- [ ] 将来の自分が文脈なしで読んで理解できる内容になっている
