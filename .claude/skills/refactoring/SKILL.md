# Refactoring Skill

## 原則

- **振る舞いを変えない**: リファクタリング中は機能追加・バグ修正を混ぜない
- **小さく進める**: 1ステップ1変更、各ステップで動作確認
- **テストが守る**: テストがないコードは先にテストを書く

## リファクタリングパターン

| パターン | 適用場面 |
|----------|----------|
| Extract Function | 長い関数・重複ロジック |
| Rename | 意図が不明な命名 |
| Remove Dead Code | 未使用の変数・関数・インポート |
| Replace Magic Number | 意味不明な数値リテラル |
| Simplify Condition | 複雑な条件式 |
| Extract Variable | 複雑な式の中間値 |

## 手順

1. 対象コードを `Read` で確認する
2. テストが存在するか確認し、なければ先に作成する
3. リファクタリング対象を1つ選ぶ
4. 変更を加える
5. テストを実行して通過を確認する（`Bash`）
6. 次の対象へ進む

## チェックリスト

- [ ] テストが全件パスしている（変更前・変更後）
- [ ] 型チェックがパスしている
- [ ] 変更が1つの懸念事項に絞られている
- [ ] コメントではなくコードが意図を語っている

## 出力フォーマット

```
## リファクタリング結果

### 変更内容
- ファイル名:行番号 — <パターン名>: <変更の説明>

### テスト結果
<テスト実行結果の概要>
```
