# Code Review Skill

## レビュー観点

1. **正確性**: バグ・ロジックエラー・エッジケース
2. **セキュリティ**: インジェクション・認証・機密情報露出
3. **品質**: 命名・重複・不要な複雑性
4. **テスト**: カバレッジ・エッジケース

## 手順

1. `git diff` で変更差分を確認
2. 各変更ファイルを `Read` で詳細確認
3. 問題点をリストアップ（重大度: HIGH / MEDIUM / LOW）
4. 修正提案を具体的に提示

## 出力フォーマット

```
## レビュー結果

### [HIGH] ファイル名:行番号
問題の説明と修正提案

### [MEDIUM] ...
```
