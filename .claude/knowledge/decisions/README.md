# Architecture Decision Records (ADR)

設計・技術選択の意思決定を記録する。「なぜそうしたか」を未来の自分と Claudeに伝える。

## ファイル命名規則

```
ADR-0001-<短いタイトル>.md
```

## テンプレート

新しいADRは以下フォーマットで作成する:

```markdown
# ADR-NNNN: タイトル

- **日付**: YYYY-MM-DD
- **状態**: 提案 | 承認 | 廃止

## 背景

なぜこの決定が必要だったか。

## 決定

何を選んだか。

## 理由

なぜその選択をしたか。却下した代替案も記載。

## 結果

この決定によって生じること（良い点・悪い点）。
```
