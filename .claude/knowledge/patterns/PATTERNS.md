# Patterns Library

実際の作業から蒸留した、再利用可能なパターン集。

## パターンの形式

```
### [パターン名]
**状況**: どんな場面で使うか
**解決策**: 具体的な手順またはコードスニペット
**注意**: 落とし穴・制約・代替案
**出典**: どのタスク/セッションから学んだか
```

---

## Git / バージョン管理

### 安全なforce-push回避
**状況**: リモートの履歴を守りながらローカル変更を反映したい
**解決策**: `git push --force-with-lease` を使う（他者の変更があれば失敗する）
**注意**: `--force` は絶対に使わない
**出典**: workspace初期設定

---

## Claude Code / 自動化

### Stopフックでの自動コミット
**状況**: セッション終了時に作業内容を確実に保存したい
**解決策**: `.claude/hooks/auto-push.sh` でdiff確認→コミット→プッシュ
**注意**: `set -euo pipefail` を使い、pushエラーは `|| true` で握りつぶす（hook失敗でセッション終了が邪魔されないように）
**出典**: workspace初期設定
