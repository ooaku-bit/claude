# Claude Code Workspace

## プロジェクト概要
Claude Code の自動化・運用ワークスペース。

## アーキテクチャ / Architecture

```
/
├── CLAUDE.md            # プロジェクトルール（このファイル）
├── .claude/
│   ├── settings.json    # 権限・フック設定（gitignored）
│   ├── skills/          # スキル定義
│   ├── commands/        # カスタムコマンド
│   └── agents/          # エージェント定義
└── .gitignore
```

## ワークフロールール / Workflow Rules

- **Plan Modeを使う**: 調査と実装を明確に分離する
- **小さく実装してすぐ検証**: 小さく → 検証 → 改善を繰り返す
- **TodoリストでWork管理**: 優先順位と進捗を常に明確に
- **Commit頻繁に**: 一度に1タスクに集中し、完了したらすぐコミット
- **ブランチ戦略**: `main` / `feature/*` / `spike/*` を使い分ける
- **/compactを定期的に**: コンテキストを圧縮して余白を確保

## コンテキスト管理

- 重要な情報・ルール・約束事は必ずこのCLAUDE.mdに書く
- サブフォルダには `.claude/` や `CLAUDE.md` を追加してローカルコンテキストを追加
- 各ファイルは200行未満に保つ
- `/compact` は戻せないので事前にCLAUDE.mdへ退避

## スペシャリスト組織 / Specialist Team

### Agents（サブエージェント）

| エージェント | 役割 | 起動タイミング |
|-------------|------|--------------|
| `architect` | システム設計・実装計画 | 新機能・大規模変更の前 |
| `debugger` | バグ根本原因解析・修正 | エラー発生時・再現困難なバグ |
| `devops` | CI/CD・デプロイ自動化 | パイプライン設計・インフラ変更 |
| `security-reviewer` | OWASP準拠のセキュリティ検査 | PR前・機密データ変更時 |

### Skills（スラッシュコマンド）

| スキル | 役割 | 使用例 |
|--------|------|--------|
| `/code-review` | 変更差分のレビュー | PR作成前 |
| `/testing` | テスト作成・実行 | 新機能・バグ修正後 |
| `/refactoring` | 安全なコード改善 | 品質向上・技術的負債解消 |
| `/performance` | ボトルネック特定・最適化 | 速度改善・負荷対策 |
| `/deploy` | デプロイ前チェック・実行 | リリース時 |

## セキュリティ

- `.env` ファイルや認証情報は絶対にコミットしない
- 外部副作用（DB操作・API呼び出し）は必ず確認してから実行
- `sudo` コマンドは原則禁止
